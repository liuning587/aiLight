> 用 ESP32-C3 + 三色交通灯，把 **Cursor / TRAE** Agent 的 **思考 / 忙碌 / 等待 / 完成 / 出错** 映射到物理灯光。  
> 设计对标 [PromLight](https://light.ghostyu.com/ai.txt)。

## 1. 工作原理

```
┌─────────────────┐     HTTP      ┌──────────────┐    BLE     ┌─────────────┐
│  Cursor / TRAE  │ ────────────► │    lightd    │ ────────► │  ESP32-C3   │
│  (项目 Hooks)   │  127.0.0.1    │  状态机+队列  │  长连接   │  三色交通灯  │
└─────────────────┘    :7801      └──────┬───────┘           └─────────────┘
                                         │
                                  Web 控制台
                              扫描 / 绑定 / 调试
```

| 组件 | 路径 | 作用 |
|------|------|------|
| 板端固件 | `firmware/main.py` | MicroPython + BLE UART，执行灯效命令 |
| 守护进程 | `tools/lightd/` | 聚合 IDE 事件、维护状态机、BLE 长连接、HTTP API |
| IDE Hook | `tools/ailight_hook.py` | 把 Cursor/TRAE 生命周期事件转发给 lightd |
| Cursor 配置 | `.cursor/hooks.json` | Cursor 项目级 Hook 入口 |
| TRAE 配置 | `.trae/hooks.json` | TRAE 项目级 Hook 入口 |
| 守护进程配置 | `config.json` | 端口、超时、各状态对应的 BLE 命令 |
| 设备绑定 | `devices.json` | MAC 地址与别名（Web 绑定后自动写入） |

**要点：**

- IDE 不直接连蓝牙，统一走本机（或局域网）的 **lightd**。
- lightd 对多个 Hook 事件做 **状态聚合**（例如同时有「忙碌」和「等待」时，优先显示等待红灯）。
- 一台电脑通常只跑 **一个** lightd 实例（端口 `7801`），多个 Cursor 窗口可共用。

## 2. 你需要准备什么

### 硬件

- ESP32-C3 SuperMini（或兼容板）
- 三色交通灯模块（共阴/共阳需与固件 `active_high` 一致，默认 **高电平点亮**）
- USB 数据线（供电 + 初次烧录）

### 软件（接灯 / 跑 lightd 的电脑）

- Windows 10/11（一键脚本 `install.ps1` 针对 Windows；macOS/Linux 可手动安装）
- Python 3.10+（需能执行 `python` 命令）
- 电脑 **蓝牙** 可用（双机部署时，仅 lightd 所在电脑需要蓝牙）
- 烧录工具：`mpremote`（`pip install mpremote`）或 Thonny 等

### IDE

- **Cursor**（启用项目 Hooks）或 **TRAE IDE**（Hooks 选「本地自动运行」）

## 3. 硬件接线

默认接线（与 `firmware/main.py` 中 `TrafficLight` 一致）：

| 灯模块 | ESP32-C3 |
|--------|----------|
| GND | GND |
| R（红） | GPIO4 |
| Y（黄） | GPIO3 |
| G（绿） | GPIO2 |

若你的模块是低电平点亮，可在固件中把 `active_high=True` 改为 `False`，或通过 BLE 命令 `POLARITY LOW` 切换（需固件支持）。

上电后，板子 BLE 广播名一般为 **`aiLight-XXXX`**，其中 `XXXX` 为 MAC 地址后四位的十六进制（例如 `aiLight-BD92`）。

## 4. 固件烧录

### 4.1 安装 MicroPython

若板子尚未刷入 MicroPython，请按 [MicroPython ESP32-C3 文档](https://docs.micropython.org/en/latest/esp32/quickref.html) 刷入对应固件。

### 4.2 上传 `main.py`

在项目根目录执行（将 `COMx` 换成实际串口，如 `COM15`）：

```powershell
pip install mpremote
mpremote connect COM15 fs cp firmware/main.py :main.py
mpremote connect COM15 reset
```

**注意：**

- 烧录/调试时 **不要** 同时让 lightd 或其它程序占用该串口。
- 上传完成后可拔掉 USB 仅用蓝牙；也可保持 USB 供电。

### 4.3 验证固件

板子上电后，用手机或电脑蓝牙扫描，应能看到 `aiLight-XXXX`。  
也可用命令行扫描（见 [§14](#14-命令行与手动控制)）。

## 5. 软件安装

### 5.1 Windows 一键安装（推荐）

在 **aiLight 项目根目录** 打开 PowerShell：

```powershell
powershell -ExecutionPolicy Bypass -File install.ps1
```

脚本会：

1. `pip install -r requirements.txt`（主要依赖 `bleak`）
2. 结束占用 `7801` 端口的旧 lightd 进程
3. 后台启动 `python -m tools.lightd`
4. 运行 Hook 连通性测试

安装成功后，浏览器打开：**http://127.0.0.1:7801**（使用说明：**http://127.0.0.1:7801/docs**）

### 5.2 手动安装（Windows / macOS / Linux）

```bash
cd /path/to/aiLight
pip install -r requirements.txt
python -m tools.lightd
```

另开终端验证：

```bash
python tools/ailight_hook.py test
# 期望输出: OK daemon | ...
```

### 5.3 开机自启（推荐：任务计划程序）

**方式 A：安装时一键注册（需已安装 Python）**

```powershell
powershell -ExecutionPolicy Bypass -File install.ps1 -Autostart
```

会在用户登录时通过 Windows **任务计划程序** 自动启动 lightd（任务名 `aiLight-lightd`）。比「启动文件夹」更稳定，且会先结束占用 7801 的旧进程再启动。

取消自启：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\register-autostart.ps1 -Unregister
```

**方式 B：手动启动脚本**

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-lightd.ps1
```

**方式 C：打包为 exe（无需目标机器安装 Python）**

在开发机执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build-lightd.ps1
```

生成 `dist\lightd.exe`。将 `lightd.exe` 与 `config.json`、`devices.json` 放在同一目录，双击或注册自启：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\register-autostart.ps1 `
  -ProjectRoot "D:\aiLight\dist" `
  -ExePath "D:\aiLight\dist\lightd.exe"
```

> exe 体积较大（含 Python 运行时 + BLE 库），一般 **有 Python 的环境优先用方式 A**；卖硬件给无 Python 用户时再用 exe。

## 6. 首次配置（Web 控制台）

打开 **http://127.0.0.1:7801**（双机部署时改为 lightd 所在机器的 IP）。

### 推荐流程

1. 确认 ESP32 已上电，电脑蓝牙已开启。
2. 点击 **「扫描附近设备」**，等待约 8 秒。
3. 在列表中找到 `aiLight-XXXX`，点击 **「绑定并使用」**。
4. 在 **「当前设备」** 区域点击 **「测试连接」**，灯应有响应。
5. 在 **「灯效调试 & 状态」** 中点「思考」「忙碌」等按钮，确认灯效正常。
6. **完全退出并重启 Cursor 或 TRAE**，新开一条 **Agent 对话**（旧对话可能不会触发 `sessionStart`）。

绑定成功后，`devices.json` 会自动写入 MAC，无需手改。

## 7. 在 Cursor 中使用

### 7.1 前提

- 用 Cursor **打开 aiLight 项目**，或在你自己的项目里配置了 Hook（见 [§13](#13-多台-cursor--多个项目共用)）。
- 项目内已有 `.cursor/hooks.json` 与 `.cursor/hooks/ailight_hook.py`。
- lightd 已运行且设备已绑定。

### 7.2 启用步骤

1. 完成 [§5](#5-软件安装)、[§6](#6-首次配置web-控制台)。
2. 在 Cursor 中确认 **Hooks 已启用**（Cursor 设置 → 搜索 Hooks，确保项目 Hook 未被禁用）。
3. **重启 Cursor**。
4. 新建 Agent 对话，发送一条消息，黄灯应进入「思考」慢闪。

### 7.3 Hook 事件映射（Cursor）

| Cursor Hook | Hook 脚本参数 | lightd 事件 | 典型场景 |
|-------------|---------------|-------------|----------|
| `sessionStart` | `session_start` | `session_start` | 新会话开始 → 灯灭 |
| `beforeSubmitPrompt` | `prompt` | `user_prompt` | 你发送消息 → 黄慢闪 |
| `preToolUse` | `tool_start` | `tool_start` | Agent 调用工具 → 黄快闪 |
| `postToolUse` | `tool_success` | `tool_success` | 工具成功 → 回到思考 |
| `postToolUseFailure` | `tool_failure` | `tool_failure` | 工具失败 → 红闪 |
| `beforeShellExecution` / `beforeMCPExecution` | `permission_wait` | `permission_wait` | 等待授权 → 红灯 |
| `afterShellExecution` / `afterMCPExecution` | `permission_done` | `permission_done` | 授权完成 |
| `stop` | `session_stop` | `session_stop` | Agent 结束 → 绿灯 60s |

配置见 `.cursor/hooks.json`。

### 7.4 验证 Hook

```powershell
python .cursor/hooks/ailight_hook.py test
python .cursor/hooks/ailight_hook.py session_start
python .cursor/hooks/ailight_hook.py prompt
```

## 8. 在 TRAE 中使用

### 8.1 启用步骤

1. 完成 lightd 安装与 Web 绑定（同 Cursor）。
2. 用 TRAE 打开本项目（或已配置 `.trae/hooks.json` 的项目）。
3. 进入 **设置 → Hooks → 项目**，确认 `.trae/hooks.json` 已加载。
4. Hook **运行方式** 选择 **「本地自动运行」**（需访问本机 `lightd`；双机部署时只需 HTTP）。
5. **重启 TRAE**，新开对话。

### 8.2 Hook 事件映射（TRAE）

| TRAE 事件 | 脚本参数 | 说明 |
|-----------|----------|------|
| `SessionStart` | `session_start` | 会话开始 |
| `UserPromptSubmit` | `prompt` | 用户发消息 |
| `PreToolUse` | `tool_start` | 工具开始 |
| `PostToolUse` | `post_tool` | 工具结束（自动区分成功/失败） |
| `Stop` | `session_stop` | 会话结束 |
| `Notification` (permission_prompt) | `notification` | 权限等待 → 红灯 |

配置见 `.trae/hooks.json`。IDE 侧连接地址在 `.trae/ailight.json`（`daemon_host` / `daemon_port`）。

## 9. 灯效与 Agent 状态对照

| 显示状态 | 灯效 | 固件命令 | 何时出现 |
|----------|------|----------|----------|
| 空闲 `idle` | 全灭 | `MODE ALL_OFF` | 会话开始、强制空闲、超时回落 |
| 思考 `thinking` | 黄慢闪 | `MODE THINK_SLOW` | 你发消息后、工具间隙 |
| 忙碌 `busy` | 黄快闪 | `MODE FLASH_YELLOW` | Agent 正在执行工具 |
| 等待 `waiting` | 红灯常亮 | `MODE WAIT` | Shell/MCP 需授权、TRAE 权限通知 |
| 完成 `done` | 绿灯常亮 | `MODE DONE` | Agent 结束（默认 **60 秒** 后自动灭） |
| 出错 `error` | 红灯闪 | `BLINK RED 6 250` | 工具失败（默认显示 **4 秒**） |

**多状态并存时的优先级**（高 → 低）：

```
出错 > 等待 > 忙碌 > 思考 > 完成 > 空闲
```

多个 Cursor 窗口或多个 Agent 同时工作时，灯会反映 **优先级最高** 的状态。

## 10. Web 控制台说明

地址：**http://127.0.0.1:7801**（或 lightd 所在主机 IP）。

**在线使用说明：** **http://127.0.0.1:7801/docs**（控制台右上角「使用说明」入口，内容与 `docs/使用说明.md` 同步）。

| 区域 | 功能 |
|------|------|
| **当前设备** | 显示默认设备名称、MAC、连接状态；**测试连接** 发送探测命令 |
| **灯效调试 & 状态** | 实时显示 `phase`；手动触发各灯效（不经过 IDE） |
| **添加 aiLight** | BLE 扫描 → **绑定并使用**（写入 `devices.json` 并设为默认） |
| **已绑定设备** | 列表管理；可切换默认、删除 |

页面会定期轮询 `/api/status` 刷新状态。

## 11. 配置文件详解

### 11.1 `config.json`（lightd 守护进程）

运行在 **接灯 / 有蓝牙** 的电脑上。

```json
{
  "web_host": "127.0.0.1",
  "web_port": 7801,
  "dry_run": false,
  "done_timeout_sec": 60,
  "waiting_timeout_sec": 300,
  "error_display_sec": 4,
  "default_device": "lab-main",
  "devices_config": "devices.json",
  "state_commands": {
    "idle": "MODE ALL_OFF",
    "thinking": "MODE THINK_SLOW",
    "busy": "MODE FLASH_YELLOW",
    "waiting": "MODE WAIT",
    "done": "MODE DONE",
    "error": "BLINK RED 6 250"
  }
}
```

| 字段 | 说明 |
|------|------|
| `web_host` | 监听地址。本机 `127.0.0.1`；局域网远程访问改为 `0.0.0.0` |
| `web_port` | HTTP 端口，默认 `7801` |
| `dry_run` | `true` 时不发 BLE，仅打日志（调试状态机） |
| `done_timeout_sec` | 完成后绿灯保持秒数 |
| `waiting_timeout_sec` | 等待状态最长保持秒数（防卡死） |
| `error_display_sec` | 出错红闪显示秒数 |
| `default_device` | `devices.json` 中的设备别名 |
| `state_commands` | 各逻辑状态对应的 BLE 文本命令 |

修改 `config.json` 后需 **重启 lightd**。

### 11.2 `.cursor/ailight.json` / `.trae/ailight.json`（IDE Hook）

运行在 **写代码的电脑** 上（可与 lightd 同机）。

```json
{
  "enabled": true,
  "default_device": "lab-main",
  "devices_config": "devices.json",
  "daemon_host": "127.0.0.1",
  "daemon_port": 7801,
  "profile": "promlight"
}
```

| 字段 | 说明 |
|------|------|
| `enabled` | 是否启用（保留字段，逻辑以 Hook 配置为准） |
| `daemon_host` | lightd 地址。本机 `127.0.0.1`；分机填 lightd 电脑局域网 IP |
| `daemon_port` | 与 `config.json` 的 `web_port` 一致 |
| `default_device` | Hook 直连 BLE 回退时使用的设备别名 |
| `devices_config` | 设备列表路径（相对 aiLight 项目根） |

修改后需 **重启 IDE**。

### 11.3 `devices.json`（设备绑定）

由 Web 控制台维护，也可手动编辑：

```json
{
  "default_timeout": 8.0,
  "default_device": "lab-main",
  "devices": {
    "lab-main": {
      "name": "aiLight-BD92",
      "name_prefix": "aiLight",
      "address": "88:56:A6:60:BD:92",
      "timeout": 8.0
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `devices.<别名>.address` | BLE MAC，绑定后自动填入 |
| `devices.<别名>.name` | 广播名，扫描时显示 |
| `name_prefix` | 扫描过滤前缀，默认 `aiLight` |

**维护位置：** 始终在 **lightd 所在电脑** 的项目目录下。

## 12. 双机部署（灯和电脑分机）

适用场景：**A 电脑接灯 + 蓝牙**，**B 电脑写代码 + Cursor**。

```
电脑 B (Cursor Hook)  ──HTTP──►  电脑 A (lightd + BLE)  ──►  ESP32
```

### 电脑 A（灯旁，例如 192.168.1.100）

1. 克隆 aiLight，运行 `install.ps1` 或 `python -m tools.lightd`。
2. 修改 `config.json`：
   ```json
   { "web_host": "0.0.0.0", "web_port": 7801 }
   ```
3. **Windows 防火墙** 放行入站 TCP **7801**。
4. 浏览器打开 `http://192.168.1.100:7801` → 扫描绑定设备。

### 电脑 B（Cursor / TRAE）

1. 克隆项目（或仅复制 Hook 相关文件，见 [§13](#13-多台-cursor--多个项目共用)）。
2. 修改 `.cursor/ailight.json`：
   ```json
   { "daemon_host": "192.168.1.100", "daemon_port": 7801 }
   ```
3. 重启 IDE。**B 机不需要蓝牙**，Hook 只发 HTTP。

> **安全提示：** `web_host: 0.0.0.0` 时，局域网内任何能访问 7801 的机器都可控灯，**无鉴权**。仅建议在可信内网使用。

## 13. 多台 Cursor / 多个项目共用

### 13.1 同一台电脑、多个 Cursor 窗口

| 场景 | 是否可用 |
|------|----------|
| 多个窗口都打开 **aiLight 项目** | ✅ 直接可用，共用一盏灯、一个 lightd |
| 多个 Agent 同时运行 | ✅ 状态机自动合并，显示优先级最高的灯态 |
| 其它项目文件夹 | ⚠️ 需单独配置 Hook（见下） |

### 13.2 在其它项目中启用 aiLight

Cursor Hook 是 **按项目目录** 生效的。要在任意仓库里用灯，有两种方式：

**方式 A：把 aiLight 作为子目录或 monorepo 一部分**  
在含 `.cursor/hooks.json` 的目录用 Cursor 打开即可。

**方式 B：在目标项目添加 Hook，指向 aiLight 安装路径**

假设 aiLight 安装在 `D:\OpenSource\aiLight`，在目标项目创建 `.cursor/hooks.json`（事件列表可复制 aiLight 仓库内的文件），将命令改为绝对路径，例如：

```json
{
  "version": 1,
  "hooks": {
    "sessionStart": [
      {
        "command": "python D:\\OpenSource\\aiLight\\.cursor\\hooks\\ailight_hook.py session_start",
        "timeout": 10
      }
    ]
  }
}
```

其余事件（`stop`、`beforeSubmitPrompt`、`preToolUse` 等）同样改为该路径。  
可选：在目标项目放 `.cursor/ailight.json`，设置 `daemon_host` / `daemon_port`。

**注意：** Hook 脚本内部会以 **aiLight 项目根** 为 `PROJECT_ROOT` 读取 `config.json` 与 `devices.json`，因此 lightd 与设备绑定仍只需在 aiLight 目录维护一份。

## 14. 命令行与手动控制

### 14.1 守护进程

```bash
# 启动（前台）
python -m tools.lightd

# Hook 连通测试
python tools/ailight_hook.py test
```

### 14.2 直接 BLE 控制（不经过 lightd）

```bash
# 列出已配置设备
python tools/ble_lightctl.py --list-devices

# 扫描 aiLight 设备
python tools/ble_lightctl.py --scan

# 对默认设备发命令
python tools/ble_lightctl.py --device lab-main --cmd "STATUS"
python tools/ble_lightctl.py --device lab-main --cmd "MODE FLASH_YELLOW"
python tools/ble_lightctl.py --device lab-main --cmd "MODE ALL_OFF"

# 自然语言（中文）
python tools/ble_lightctl.py --nl "黄灯闪"
python tools/ble_lightctl.py --nl "全灭"
```

### 14.3 板端支持的 BLE 文本命令

通过 Nordic UART 服务发送一行文本（以换行结束）：

| 命令 | 说明 |
|------|------|

## 17. 故障排查

### Hook 完全不亮

1. 浏览器能否打开 `http://127.0.0.1:7801`（或远程 lightd 地址）？
2. 运行 `python tools/ailight_hook.py test` 是否 `OK daemon`？
3. Web 控制台 **测试连接** 是否成功？
4. 是否 **重启过 IDE** 并 **新开 Agent 对话**？
5. `.cursor/ailight.json` 中 `daemon_host` / `daemon_port` 是否正确？

### 双机部署不亮

1. A 机 `config.json` 中 `web_host` 是否为 `0.0.0.0`？
2. A 机防火墙是否放行 TCP 7801？
3. B 机能否浏览器访问 `http://<A机IP>:7801`？
4. B 机 `daemon_host` 是否为 A 机局域网 IP（不是 `127.0.0.1`）？

### TRAE 不亮

1. Hooks 是否选 **「本地自动运行」**？
2. 设置 → Hooks → **运行日志** 是否有报错？
3. 是否使用 **新开对话** 测试？

### BLE 连不上 / 扫描不到

1. 板子是否上电？广播名是否为 `aiLight-XXXX`？
2. 串口是否被 `mpremote`、Thonny 占用？关闭后 `mpremote reset`。
3. 电脑蓝牙是否开启？与其它 BLE 调试工具是否冲突？
4. `devices.json` 中 MAC 是否与当前板子一致（换板需重新绑定）？

### 灯态异常

| 现象 | 处理 |
|------|------|
| 空闲时红绿同亮 | 升级最新固件（`SET` 互斥 + `ALL_OFF` 清状态） |
| 切换很慢 | 确认 lightd 在跑且为长连接版本；重启 lightd |
| Web「思考」按钮无效 | 确认 lightd 已更新（支持 `thinking` 事件） |
| 7801 端口被占用 | `install.ps1` 会自动杀旧进程；或手动结束占用进程 |

### Hook 超时

- 检查 lightd 是否卡死；查看终端/lightd 日志。
- 适当增大 `.cursor/hooks.json` 中 `timeout`（默认 8–10 秒）。
- 确认 BLE 连接正常，避免守护进程阻塞。

## 18. HTTP API 参考

Base URL：`http://<host>:<port>`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | Web 控制台页面 |
| GET | `/docs` | 使用说明（Markdown 渲染页） |
| GET | `/api/docs` | 使用说明原文 JSON `{title, content}` |
| GET | `/api/status` | 当前 phase、BLE 状态、状态机详情 |
| GET | `/api/config` | 读取 `config.json` |
| POST | `/api/config` | 更新配置（JSON body） |
| POST | `/api/event` | 发送事件，body: `{"event":"thinking"}` |
| GET | `/api/devices` | 已绑定设备列表 |
| POST | `/api/devices/scan` | BLE 扫描，body: `{"timeout":8}` |
| POST | `/api/devices/bind` | 绑定，body: `{"address":"..","name":"..}` |
| POST | `/api/devices/activate` | 切换默认设备，body: `{"alias":"lab-main"}` |
| POST | `/api/devices/test` | 测试连接 |
| DELETE | `/api/devices/<alias>` | 删除设备 |

**事件名**（用于 `/api/event`）：  
`session_start`, `user_prompt`, `tool_start`, `tool_success`, `permission_wait`, `permission_done`, `session_stop`, `tool_failure`, `force_idle`, `thinking`, `busy`, `waiting`（后三个供 Web 调试）。

示例：

```bash
curl -X POST http://127.0.0.1:7801/api/event -H "Content-Type: application/json" -d "{\"event\":\"thinking\"}"
```

## 附录：与 PromLight 的差异

- 需自行烧录 MicroPython 与 `main.py` 固件
- 无成品外壳、按键、续航管理
- 灯效为三色闪/常亮（无跑马灯等复杂效果）
- 官方 Hook 仅内置 **Cursor** 与 **TRAE** 项目配置
- HTTP API 无鉴权（内网自用）

**快速入口：** 安装后打开 http://127.0.0.1:7801 ，扫描绑定，重启 IDE，新开 Agent 对话即可。
